## 一、项目概览

这是一个基于 Node.js 的本地 Web 应用，用于从海量照片中精选、自动排版并导出为可打印的相册。

- **运行模式**: 浏览器模式（通过 `server.js` 启动本地 HTTP 服务）
- **开发语言**: 前端 HTML/CSS/JS，后端 Node.js
- **核心文件**: `app-browser.js`（约 120KB，包含所有业务逻辑）

## 五、核心数据流

### 5.1 照片扫描

```
用户选择目录 → server.js /api/scan → 递归扫描
  → 按年份/月份归类 → 返回 JSON → 前端渲染导航树
```

### 5.2 照片精选

```
用户点击照片 → 加入 state.selectedPhotos (Set)
  → 右侧"已精选"面板更新
```

### 5.3 自动排版

```
用户点击"自动排版" → autoLayout()
  → 按年份分组 → 每页1-4张 → 生成 state.albumPages[]
  → renderAlbumPages() 渲染到 #album-pages 容器
```

### 5.4 导出流程

```
用户点击"导出打印" → 选择纸张尺寸/格式
  → html2canvas 截取每页 → jsPDF 合成 PDF
  → 浏览器下载
```

## 八、2026-06-10 修复记录

### 🔧 问题 1: 装饰素材无法拖入页面

**症状**: 从装饰素材面板拖拽 Emoji/SVG/边框到排版页面，没有任何反应。

**根本原因**: Chrome 浏览器在 `dragover` 事件中**禁止读取** `e.dataTransfer.getData()` 的自定义数据（安全策略），导致 `dropEffect` 设置错误。

**修复详情**:

#### 修复 1.1: `dragover` 改用 `types` 判断

**文件**: `app-browser.js`  
**位置**: `renderAlbumPages()` 中的页面 `dragover` 事件监听器

```js
// 修复前（错误）：dragover 中 getData 永远返回空字符串
const decorType = e.dataTransfer.getData('application/decor-type');
if (decorType) {
  e.dataTransfer.dropEffect = 'copy';
} else {
  e.dataTransfer.dropEffect = 'move';
}

// 修复后（正确）：用 types 数组判断
if (e.dataTransfer.types.includes('application/decor-type')) {
  e.dataTransfer.dropEffect = 'copy';
} else {
  e.dataTransfer.dropEffect = 'move';
}
```

**原理**: 
- `e.dataTransfer.types` 在 `dragover` 中可用，返回所有设置的 MIME 类型数组
- `e.dataTransfer.getData()` 在 `dragover` 中被 Chrome 屏蔽，只在 `drop` 事件中可用
- 装饰素材 `effectAllowed = 'copy'`，页面 `dropEffect = 'move'` 不匹配 → 浏览器拒绝 drop

#### 修复 1.2: 无页面时添加全局 drop zone

**文件**: `app-browser.js`  
**位置**: 初始化部分（约第 2915 行），新增全局 drop 监听

当用户还没有点击"自动排版"（`albumPages` 为空）时，页面区域没有 `.album-page` 元素，无法接收 drop 事件。新增 `#album-pages` 容器的全局 drop 监听：

```js
const albumPagesContainer = $('#album-pages');
albumPagesContainer.addEventListener('dragover', (e) => {
  if (e.dataTransfer.types.includes('application/decor-type')) {
    e.preventDefault();
    e.dataTransfer.dropEffect = 'copy';
    albumPagesContainer.classList.add('drag-over');
  }
});
albumPagesContainer.addEventListener('drop', (e) => {
  // 自动创建空白页面，然后添加装饰元素
  if (state.albumPages.length === 0) {
    state.albumPages.push({ type: 'free', elements: [] });
    renderAlbumPages();
  }
  // 延迟等 DOM 更新后添加装饰...
});
```

#### 修复 1.3: CSS 拖拽反馈

**文件**: `styles.css`  
**位置**: `.album-pages` 样式块

新增 `.album-pages.drag-over` 样式，拖拽装饰素材到页面区域时显示蓝色虚线边框。

#### 修复 1.4: 点击装饰自动创建页面

**文件**: `app-browser.js`  
**位置**: `addDecorToPage()` 函数

原来点击装饰时，如果 `albumPages` 为空则直接提示"请先完成排版"。修改为：如果有已精选的照片，自动创建一个空白自由排版页面，然后再添加装饰。

### 🔧 问题 4: 刷新页面导致排版数据丢失（自动保存）

**时间**: 2026-06-10 20:56  
**症状**: 用户在排版视图编辑照片、素材后，误点浏览器刷新或关闭标签页，所有排版数据丢失，变成空白。

**根本原因**: 所有数据存储在内存 `state` 对象中，没有持久化机制。

**修复方案**: 实现三层防护

#### 4.1 localStorage 自动保存

- 新增 `autoSave()` 函数（300ms 防抖），在以下时机自动保存完整状态到 `localStorage`：
  - 排版页面每次渲染完毕（`renderAlbumPages()` 末尾）
  - 照片精选/取消精选（`toggleSelectPhoto`、全选、清空、移除精选）
- 保存数据包含：`rootDir`、`selectedPhotos`、`albumPages`、`theme` 等所有关键状态
- 保存 key：`photo-album-autosave`

#### 4.2 启动时自动恢复

- 新增 `checkAutoSave()` — 检测 localStorage 是否有自动保存数据
- 新增 `restoreAutoSave(data)` — 恢复排版状态并切换到排版视图
- 页面加载后弹出确认对话框，显示：
  - 精选照片数量
  - 排版页面数量
  - 保存时间（刚刚/X分钟前/X小时前）
- 用户点"确定"恢复，点"取消"清除自动保存

#### 4.3 beforeunload 警告

- 监听 `window.beforeunload` 事件
- 当存在未保存的排版内容时，刷新/关闭页面弹出浏览器原生确认提示
- 提示文字："您有未保存的排版内容，确定要离开吗？建议先点击"保存"按钮。"

#### 4.4 手动保存优化

- 点击"保存"按钮后清除 `_hasUnsavedChanges` 标记
- 保存按钮 toast 提示增加"已下载 .album 文件"说明

**修改文件**: `app-browser.js`（末尾初始化部分，约 80 行新代码 + 5 处 autoSave() 调用点）

### 🔧 问题 5: 三大体验增强（智能评分 + 对齐辅助线 + 键盘快捷键）

**时间**: 2026-06-10 21:08  

#### 5.1 照片智能评分推荐

**原理**：根据照片文件大小和文件名关键词自动打分（1-5星），减少选片工作量。

- **评分规则**：
  - 文件 > 10MB → +2分；> 5MB → +1分；< 1MB → -1分；< 0.3MB → -2分
  - 文件名含 `best/精选/封面/cover/main` → +1分
  - 文件名含 `blur/模糊/temp/临时/截图/screenshot` → -2分
  - 文件名含 `thumb/thumbnail/缩略图` → -2分
- **显示**：每张照片卡片左下角显示 ⭐ 评分徽章（1-5星），hover 显示文件大小
- **排序按钮**：`⭐ 按评分排序` — 点击后高分照片优先排列，再次点击恢复时间排序
- **缓存**：评分结果缓存在 `photoScoreCache` 中，避免重复计算

#### 5.2 对齐辅助线

**原理**：拖拽元素时自动检测与其他元素和画布中线的对齐关系，显示粉色辅助线并吸附。

- **检测对齐**：左/右/中心（X方向）+ 上/下/中心（Y方向）
- **吸附阈值**：6px 内自动吸附
- **参考对象**：其他元素的边缘和中心 + 画布中线
- **视觉效果**：粉色半透明辅助线（`#ff4081`），垂直/水平方向
- **实现**：`getAlignGuides()` + `showAlignGuides()` + `hideAlignGuides()`

#### 5.3 键盘快捷键

| 快捷键 | 功能 | 视图 |
|--------|------|------|
| `Ctrl+S` | 保存项目 | 全局 |

- **撤销系统**：`undoStack` 数组，深拷贝 `albumPages`，最多保存 50 个历史状态
- **智能快照**：拖拽开始、应用模板前自动推入撤销栈；相同状态不重复推入

### 🔧 问题 6: 照片元素无法在同一页面内拖拽移动位置

**时间**: 2026-06-10 21:42  
**症状**: v3.7 更新后，已排版的照片无法在页面内自由拖拽摆放位置，拖拽后弹回原位。

**根本原因**: 照片元素使用原生 HTML5 拖拽（`dragstart`/`drop`），但 `handleDropOnPage()` 函数只处理了两种情况：
1. 从素材库拖入新照片（有 `photoPath`，无 `elementId`）
2. 从其他页面拖入（`elementId` 存在且 `fromPageIndex !== pageIndex`）

**缺失了情况3**：同一页面内拖拽移动位置（`elementId` 存在且 `fromPageIndex === pageIndex`），drop 事件被忽略，照片弹回原位。

**修复**: 在 `handleDropOnPage()` 中新增情况3的处理逻辑：
- 当 `elementId` 存在且 `fromPageIndex === String(pageIndex)` 时
- 直接从 `page.elements` 中找到对应元素
- 根据鼠标放置位置更新 `el.x` 和 `el.y`
- 重新渲染页面

**修改文件**: `app-browser.js`（`handleDropOnPage` 函数，新增约 12 行）

### 9.1 Chrome 拖拽 API 关键知识

| 事件 | `getData()` | `types` | `setData()` |
|---|---|---|---|
| `dragstart` | ✅ 可读写 | ✅ | ✅ |
| `dragover` | ❌ 被屏蔽 | ✅ | ❌ |
| `drop` | ✅ 可读 | ✅ | ❌ |

**规则**: 在 `dragover` 中永远不要用 `getData()` 读取自定义类型，用 `types.includes()` 替代。

### 9.2 `dropEffect` 与 `effectAllowed` 匹配规则

- `dropEffect` 必须与 `effectAllowed` 兼容，否则浏览器拒绝 drop
- 装饰素材: `effectAllowed = 'copy'` → 接收方 `dropEffect = 'copy'`
- 照片素材: `effectAllowed = 'move'` → 接收方 `dropEffect = 'move'`

### 9.3 缓存问题

修改 `app-browser.js` 或 `styles.css` 后：
- 先 `Ctrl+Shift+R` 强制刷新
- 如果不行，关闭标签页，杀掉 node 进程，重新启动

### 9.4 端口固定

`server.js` 使用 `PORT = 3456`（固定端口），每次启动无需查看控制台，直接访问 `http://localhost:3456`。

### 9.5 项目保存/加载与自动保存

- **自动保存**: 所有排版操作自动存入浏览器 `localStorage`，刷新页面不会丢失
- **手动保存**: 点击"保存"按钮下载 `.album` JSON 文件到本地（完整备份）
- **手动打开**: 点击"打开"按钮加载之前下载的 `.album` 文件
- **启动恢复**: 每次打开网页，如有自动保存数据，弹出恢复确认对话框
- **关闭提醒**: 有未保存内容时刷新/关闭页面会弹出确认提示
